Setup de administrador do BRASFLIX

Esta página foi ajustada para o seu cenário: sem terminal local, com configuração manual pelo navegador, Firebase Console, Vercel e GitHub.

Importante: no projeto atual, o painel admin valida o campo role no documento usuarios/{uid}. As custom claims são apenas sincronizadas depois, para reforço no backend. Então o passo principal é garantir o role: "admin" no Firestore.

Fluxo recomendado

  1. Crie sua conta normalmente em login.html.
  2. No Firebase Console, abra Firestore Database.
  3. Entre na coleção usuarios e abra o documento do seu UID.
  4. Crie ou ajuste o campo role com o valor exato admin.
  5. Salve, faça logout e login novamente.
  6. Acesse /admin/dashboard.html ou /admin/dashboard.
Dica: depois que um admin válido entra no painel, o front chama /api/admin-sync-claims com o token do usuário. Isso permite espelhar a role em custom claims no Firebase Admin sem você precisar usar terminal local.

Se o documento do usuário ainda não existir

  • Faça login pelo menos uma vez via e-mail/senha ou Google.
  • O front deve criar usuarios/{uid} automaticamente.
  • Se ainda assim não aparecer, revise as regras do Firestore e as variáveis do Firebase na Vercel.

Checklist rápido

  • Authentication com Email/Senha e Google ativados
  • Domínio da Vercel em Authorized domains do Firebase Auth
  • Documento em usuarios/{uid} com role: "admin"
  • Variáveis FIREBASE_ADMIN_* configuradas na Vercel
